# Your Complete Workflow

## Step 1: Identify the Task Type

- New feature implementation
- Modify existing code
- Bug fix
- Optimization
- Test generation

## Step 2: Select Relevant Structure Elements

- All tasks: `<task>`, `<requirements>`, `<constraints>`
- Modify existing: Add `<existing_code>`
- Complex logic: Add `<thinking>`
- Pattern matching: Add `<example>`
- Library concerns: Add allowed/forbidden libraries

## Step 3: Populate Structured Prompt

- Write clear task description
- Provide necessary context
- List explicit requirements
- Set appropriate constraints
- Include examples from your codebase

## Step 4: Generate and Review

- Get AI output
- Apply Lesson 1 universal checklist
- Apply Lesson 2 pattern checklist
- Verify against structured requirements

## Step 5: Iterate if Needed

- Refine structure based on results
- Add missing constraints or examples
- Clarify ambiguous requirements
